# RUNBOOK — 운영자 실행 절차(초보용)

## 0) 운영자가 알아야 할 “원리 1줄”
- 고객이 `index.html`에 입력 → Notion DB에 접수 생성
- 출고 준비가 되면 Notion에서 처리상태를 “출고준비”로 변경
- CJ 엑셀을 `tracking.html`에 넣고 “미리보기 → 적용”하면
  출고준비 건에 송장번호가 자동 입력됨(옵션으로 출고완료 자동 전환 가능)

---

## 1) 고객 배송정보 입력(고객용)
1. 고객에게 링크 안내  
   - https://notion-address-form.vercel.app/index.html
2. 고객이 입력 완료하면 “접수번호(접수증)”가 생성됨  
3. 고객에게 안내(권장)  
   - 접수번호를 복사/저장  
   - 접수 조회는 https://notion-address-form.vercel.app/status.html

---

## 2) 접수 조회(고객용/운영자 확인용)
1. 접수 조회 링크 접속  
   - https://notion-address-form.vercel.app/status.html
2. 접수번호 입력 → 조회
3. 안내
   - 출고완료 + 송장번호가 있어야 송장번호 표시됨

---

## 3) 출고 준비(노션에서 해야 하는 것)
송장 반영은 “출고준비”만 대상으로 합니다.

1. Notion DB(출고 접수 관리)에서 해당 주문(접수번호)을 찾습니다.
2. 처리상태를 **출고준비**로 바꿉니다.
3. 여러 건 출고라면, “출고준비” 보기(View)를 만들어서 그 보기에 모아두는 것을 권장합니다.

---

## 4) 송장번호 자동 반영(운영자용)
링크:
- https://notion-address-form.vercel.app/tracking.html

### 4-1. 준비물
- CJ대한통운 엑셀(송장번호 포함)
- 운영자 비밀번호(TRACKING_ADMIN_PASS)

### 4-2. 실행 순서(정석)
1) 운영자 비밀번호 입력  
2) CJ 엑셀 파일 선택(또는 드래그&드롭)  
3) “엑셀 로드 완료(몇 건)” 표시 확인  
4) **미리보기** 클릭  
   - 결과 요약 확인: 일치/미일치/중복/업데이트/건너뜀
5) 문제 없으면 **적용** 클릭  
6) Notion에서 2~3건 샘플로 송장번호가 들어갔는지 확인

### 4-3. 옵션 설명(운영 기준)
- 기존 송장번호도 덮어쓰기  
  - 기본 OFF 권장(실수 방지)
- 적용 시 출고완료+출고일시(오늘) 자동  
  - 운영 방식에 따라 ON/OFF
- 미일치 상태 확인(느림)  
  - 기본 OFF 권장  
  - “왜 미일치인지 원인 파악이 필요할 때만” ON

---

## 5) 자주 발생하는 문제/해결(체크리스트)

### A) “엑셀 파일을 먼저 선택/드롭 해주세요”
- 파일이 실제로 로드되지 않은 상태
- 파일명/건수 표시가 뜨는지 확인 후 미리보기 실행

### B) “엑셀 첫 행에 ‘고객주문번호’ 또는 ‘운송장번호’ 컬럼이 없습니다”
- CJ 엑셀 첫 행(헤더)이 다른 형식
- 엑셀에서 컬럼명을 확인하거나, CJ 다운로드 옵션을 표준으로 다시 받기

### C) “운영자 비밀번호가 올바르지 않습니다”
- 비밀번호 입력 오타
- Vercel 환경변수 TRACKING_ADMIN_PASS가 설정되어 있는지 확인
- 변경했으면 Redeploy 필요

### D) “일치(출고준비 매칭) 0건”
자주 이유:
1) 이미 출고완료로 바뀐 건
2) 아직 출고준비로 안 바뀐 건
3) 조회기간(최근 N일)이 너무 짧음
4) 접수번호가 다름(엑셀 고객주문번호와 노션 접수번호가 동일해야 함)

---

## 6) 운영 안전수칙(실수 방지)
- “미리보기” 없이 “적용”부터 누르지 않기
- 같은 엑셀을 중복으로 반영해도 큰 문제는 없도록 설계(덮어쓰기 OFF 기준)
- 대량 작업(수천 건)은 CHUNK/딜레이 튜닝 필요 → `CONFIG.md` 참고
- 이상하면 먼저:
  - 비번/조회기간/출고준비 상태/엑셀 컬럼명 4가지만 확인하면 대부분 해결됨
